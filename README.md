# AgenticCriticPipeline

> src→target gap critique pipeline with **dimension critics** (objective) + **soul-skill artist panel** (subjective/aesthetic)

Given a **source image** (generated by a model we're evaluating, e.g. SenseNova-U1) and a **target image** (a reference or oracle, e.g. gpt-image-2-pro), the pipeline evaluates the gap between them across 4 channels, scores the severity, and outputs structured repair directives — all powered by a VLM (currently `gemini-3-flash-preview` via an OpenAI-compatible API).

## Architecture

```
                    ┌──────────────────────────────────────┐
                    │           GapPipeline.evaluate()      │
                    │                                      │
  src_path ────────▶│  ┌──────────────────────────────┐   │

  task_type ───────▶│  │  task_type → critics select  │   │
  target_desc ─────▶│  └──────────┬──────────┬────────┘   │
                    │             │          │             │
                    │    ┌────────▼──┐  ┌───▼──────────┐  │
                    │    │ DIMENSION │  │   ARTIST     │  │
                    │    │ CRITICS   │  │   PANEL      │  │
                    │    │ (7 always)│  │ (2-3 routed) │  │
                    │    └────────┬──┘  └───┬──────────┘  │
                    │             │          │             │
                    │    ┌────────▼──────────▼─────────┐  │
```

## One Critique Round — What Happens Internally

> Below is a single `evaluate()` call, annotated step by step.

### Step 1: Route
The **Router** analyzes `task_type` (portrait / landscape / poster / abstract) and `target_descriptor` to select:
- **Dimension critics**: all 7 always run
- **Artist critics**: 2–3 artists from the pool of 6, chosen by keyword matching on domain expertise

Example: a `portrait` task routes to `caravaggio` (anatomy realist), `rutkowski` (digital concept art), and `monet` (light/atmosphere).

### Step 2: Dimension Critics (Objective)
7 critics evaluate the src→target gap, each focusing on one dimension:

| Critic | Channel | Method | What it checks |
|--------|---------|--------|----------------|
| `compositional` | STRUCTURAL | VLM score | Layout, spatial arrangement |
| `stylistic` | STYLISTIC | VLM score | Color, brushwork, lighting |
| `semantic` | SEMANTIC | VLM score | Entities, subject matter |
| `quality` | QUALITY | VLM score | Finish level, rendering |
| `palette_diff` | STYLISTIC | Programmatic | Color palette distance |
| `text_fidelity` | SEMANTIC | Programmatic | Text rendering accuracy |
| `anatomy` | QUALITY | VLM score | Anatomical plausibility |

Each critic returns a `GapScore` with `score` (0–10, snapped to {0,2,4,6,8,10}), `confidence` (0–1), and `rationale`.

### Step 3: Artist Critics (Subjective / Aesthetic)
Each routed artist receives the `src` and `target` images and a prompt that includes:
- Their **persona** (e.g. "I am Caravaggio. Light is a knife that cuts truth from shadow...")
- Their **axioms** (non-negotiable aesthetic principles, 5 per artist)
- Their **perceptual bias** (which channels they notice first)
- Instructions to evaluate all 4 channels on a 0–10 anchor scale

The VLM returns structured JSON with per-channel scores and rationale. Artists who are "on-domain" (high keyword match) get higher confidence weights.

### Step 4: Aggregate
The **Aggregator** merges dimension and artist scores:
- Per-channel merged score: weighted average of dimension + artist scores
- **Disagreement penalty**: if artists disagree too much, overall alignment is penalized (anti reward-hacking)
- **Overall alignment**: inverted 0–10 scale → 0–1 where 1 = perfect match
- **Repair directives**: sorted by priority, actionable suggestions

### Step 5: Trajectory
Every raw intermediate result is saved in a `CritiqueTrajectory`:
- Router decision + reasoning
- All 7 dimension critic raw scores
- All artist × channel raw scores (e.g., 3 artists × 4 channels = 12 evaluations)
- Aggregated result

This is the data structure used for GRPO training and post-hoc analysis.

## Gap Channels

| Channel | Description | Example Signal |
|---------|-------------|----------------|
| **STRUCTURAL** | Composition, layout, spatial relationships | Subject off-center vs centered |
| **STYLISTIC** | Color palette, brushwork, lighting, mood | Warm tones absent in target |
| **SEMANTIC** | Content, entities, what is depicted | Missing object mentioned in prompt |
| **QUALITY** | Finish level, resolution, technical execution | Blurry vs sharp rendering |

Scoring anchor: `0` = perfect alignment, `10` = completely different.

## Soul-Skill Artist Panel

Six artists with distinct aesthetic worldviews:

| Artist | Era | Perceptual Bias | Domain |
|--------|-----|-----------------|--------|
| **Caravaggio** | Baroque | QUALITY, STRUCTURAL | Portrait, figure, anatomy, dramatic light |
| **Monet** | Impressionist | STYLISTIC, QUALITY | Landscape, light, color, atmosphere |
| **Hokusai** | Ukiyo-e | STRUCTURAL, STYLISTIC | Minimalist, line art, composition |
| **Mondrian** | Abstract/De Stijl | STRUCTURAL, STYLISTIC | Geometric, grid, poster, logo |
| **Moebius** | Modern Comic | STYLISTIC, STRUCTURAL | Line art, illustration, sci-fi |
| **Rutkowski** | Digital Concept | QUALITY, STYLISTIC | Fantasy, rendering, material, game art |

Each artist has 5 non-negotiable **axioms** (e.g. Caravaggio: *"Light is a knife — it should cut the figure from darkness, not caress it"*). These are injected into the VLM prompt to produce strongly opinionated, stylistically distinct critiques.

## Quick Start

```python
from gap_critique import GapPipeline

pipeline = GapPipeline(
    vlm_api_key="sk-...",
    vlm_base_url="http://10.210.0.46:40017",
    vlm_model="gemini-3-flash-preview",
    task_type="portrait",
)

result = pipeline.evaluate(
    src_path="source.png",
    target_path="target.png",
    target_descriptor="Close portrait of an elderly woman...",
)

gap = result["gap"]
print(f"Alignment: {gap.overall_alignment:.3f}")
print(f"Artist consensus: {gap.artist_consensus:.3f}")

# Full trajectory for GRPO
trajectory = result["trajectory"]
traj_dict = trajectory.to_dict()  # JSON-serializable
```

## Backends

The VLM backend is **pluggable**. Currently supported:

| Backend | Description |
|---------|-------------|
| `OpenAICompatVLM` (real) | Any OpenAI-compatible `/v1/chat/completions` endpoint with vision |
| `MockVLM` (mock) | Deterministic hash-based scores for testing without API calls |

The pipeline falls back to `MockVLM` when no API credentials are provided, or when individual API calls fail — so it never crashes mid-evaluation.
